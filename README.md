    <summary>11. graph</summary>
    그래프 자료구조를 인접리스트를 활용하여 구현, 그래프 탐색 알고리즘과 그에 대한 응용문제 풀이

        - 예제
        1. DFS
            깊이 우선 탐색을 직접 구현함
            재귀를 활용한 방법, Stack를 활용한 방법으로 나뉘어져 있으며, 재귀를 활용해 간단하게 문제를 풀이함
            탐색, 최단경로가 아닌 대부분의 문제는 깊이 우선 탐색을 고려

        2. BFS
            너비 우선 탐색을 직접 구현함
            QUEUE를 활용한 방법으로 구현 했으며, 최단경로 문제에서 특히 많이 활용

        3. Dijkstra
            다익스트라 알고리즘을 직접 구현함
            가중치가 있는 그래프에서 최단 경로를 BFS 알고리즘을 통해 보장함
            *** 음의 가중치가 있을 경우 다익스트라 알고리즘은 최적해를 보장하지 않는다. 이 점을 유의할 것 ***
            Node라는 클래스를 도입해 dest, cost를 관리 함으로서 최단경로를 구하려 했음
            *우선순위 큐*를 도입하여 가중치를 오름차순으로 정렬 후 탐색을 이어 감

        - 응용
        4. Shortcut_in_game *<중요>*
            좌표계에서 최단거리를 보장하는 알고리즘을 구현 해야 함
            이차원 배열을 통해 좌표계를 구현 후
            Node를 통해 좌표 위치를 특정, rx ry를 통해 Node의 움직임을 간소화 시킴
            반복문을 통해 rx ry로 Node 움직임을 특정하고 맵 밖이나 벽으로 갈 경우 예외처리를 시킴
            최단거리는 BFS 알고리즘을 활용해 문제를 풀이 함
            좌표계를 다루는 기본적인 문제이므로 자주 풀이 할 것

        5. Network
            깊이 우선 탐색을 활용해 연결 되어 있는 그래프의 수를 계산하는 알고리즘 작성

        6. Escape
            4. Shortcut_in_game 문제와 흡사하나, 중간에 레버를 반드시 방문 한 후 목적지에 도달해야한다.
            최단 경로를 탐색해야하니 너비 우선 탐색을 활용하고 rx ry, Node 클래스를 활용한다
            또한 기존 BFS와 다르게 파라미터에 시작 지점과 종료 지점을 같이 넣어준다.
            이를 활용해 BFS(start, lever) BFS(lever, end) 사용하여 시작 지점에서 레버를 거쳐 종료 지점을 탐색한다

        7. delivery
            가중치 그래프의 최단 경로는 다익스트라 알고리즘을 사용한다. 다익스트라 알고리즘은 우선순위큐를 사용한다.
            dist 배열을 활용해 시작지점 부터 목적지까지 최단경로를 업데이트한다.
            큐를 poll하고 poll한 노드에 연결 되어 있는 노드를 쭉 살펴본다.
            쭉 살펴 본 노드의 dist가 시작 지점에서 현 지점의 cost의 합 보다 크다면 갱신한다.
            dist를 지속적으로 갱신하며 dist가 노드의 최단 거리를 가리키도록 한다
            우선순위 큐를 사용해 노드의 최단거리를 확정적으로 방문하는 trick을 사용한다
            그냥 큐를 사용해도 dist가 갱신되지만 게산 효율이 극히 떨어진다.

        8. transmission_tower
            int cnt = dfs(next);
            처럼 변수에 재귀 함수의 반환 값을 저장할 때 햇갈리는 부분이 있어서 정리 했다
            요지는 재귀 함수가 스택 처럼 쌓일 때 반환 값이 어떻게 처리 되는가가 이해가 안 됐다.
            가장 마지막(즉 스택의 가장 위)에 호출된 함수가 먼저 계산된다.
